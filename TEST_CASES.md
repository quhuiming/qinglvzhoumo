# 情侣周末测试用例

## 1. 测试范围

本用例覆盖本轮产品方案实现内容：

- 情侣空间状态查询、邀请码加入、退出/解绑。
- 前后端同步来源字段 `authorUserId`。
- 共享空间页状态、成员数、最近错误、同步范围说明。
- 首页“TA 的动态”、双人每日回答、纪念日提示、最近完成。
- 愿望完成后引导记录回忆。
- 旧本地数据兼容与未绑定情侣空间的单人使用。
- 后端异常处理：非法 JSON 返回 400，未知异常记录日志。
- 源码入口收口：前端源码只使用 `src/`。

## 3. 后端接口用例

### TC-BE-001 匿名设备登录

前置条件：后端服务已启动。

步骤：

1. 调用 `POST /api/auth/anonymous`。
2. 请求体：

```json
{
  "deviceId": "device-a",
  "nickname": "A"
}
```

预期结果：

- HTTP 200。
- 返回 `userId`、`token`、`nickname`。
- `nickname` 为 `A`。

### TC-BE-002 创建情侣空间邀请码

前置条件：设备 A 已登录，并取得 token。

步骤：

1. 使用 A 的 token 调用 `POST /api/couples/invite`。

预期结果：

- HTTP 200。
- 返回 `coupleId`、`inviteCode`、`memberCount`。
- `memberCount` 为 1。

### TC-BE-003 B 使用邀请码加入情侣空间

前置条件：

- 设备 A 已创建邀请码。
- 设备 B 已登录，并取得 token。

步骤：

1. 使用 B 的 token 调用 `POST /api/couples/join`。
2. 请求体：

```json
{
  "inviteCode": "A生成的邀请码"
}
```

预期结果：

- HTTP 200。
- 返回的 `coupleId` 与 A 的 `coupleId` 一致。
- `memberCount` 为 2。

### TC-BE-004 查询当前情侣空间状态

前置条件：A、B 已加入同一个情侣空间。

步骤：

1. A 调用 `GET /api/couples/me`。
2. B 调用 `GET /api/couples/me`。

预期结果：

- HTTP 200。
- A、B 返回相同 `coupleId`。
- 返回 `userId`、`nickname`、`inviteCode`、`memberCount`。
- `memberCount` 为 2。

### TC-BE-005 A 推送愿望，B 拉取同步

前置条件：A、B 已加入同一个情侣空间。

步骤：

1. A 调用 `POST /api/sync`。
2. 请求体：

```json
{
  "items": [
    {
      "type": "wish",
      "clientId": "wish-001",
      "version": 1,
      "updatedAt": "2026-04-23T00:00:00Z",
      "payload": {
        "id": "wish-001",
        "title": "周末去看展",
        "done": false
      }
    }
  ]
}
```

3. B 调用 `GET /api/sync`。

预期结果：

- A 推送 HTTP 200。
- B 拉取 HTTP 200。
- B 返回列表包含 `clientId = wish-001`。
- 返回项包含 `authorUserId`，值为 A 的 `userId`。

### TC-BE-006 B 完成愿望后 A 拉取状态

前置条件：TC-BE-005 已完成。

步骤：

1. B 调用 `POST /api/sync`，推送同一个 `clientId = wish-001`，`done = true`。
2. A 调用 `GET /api/sync`。

预期结果：

- A 拉取到该愿望。
- `payload.done` 为 `true`。
- `authorUserId` 不为空。

### TC-BE-007 退出情侣空间

前置条件：A、B 已加入同一个情侣空间。

步骤：

1. B 调用 `POST /api/couples/leave`。
2. B 调用 `GET /api/couples/me`。
3. A 调用 `GET /api/couples/me`。

预期结果：

- B 退出接口 HTTP 200。
- B 返回 `coupleId = null`，`inviteCode = null`，`memberCount = 0`。
- A 本身数据不被删除。
- A 仍可查询自己的空间状态。

### TC-BE-008 非法 JSON 返回 400

前置条件：后端服务已启动。

步骤：

1. 调用 `POST /api/auth/anonymous`。
2. 请求体传入非法 JSON，例如 `{bad-json}`。

预期结果：

- HTTP 400。
- 返回 `code = INVALID_JSON`。
- 返回 `message = 请求数据格式不正确`。

## 4. 前端共享空间用例

### TC-FE-001 未绑定空间时查看共享空间

前置条件：清空本地缓存或使用新浏览器环境。

步骤：

1. 打开 H5。
2. 进入“我的/资料/共享空间”区域。

预期结果：

- 显示未绑定状态。
- 可看到创建邀请码或加入邀请码入口。
- 同步状态不阻塞本地使用。

### TC-FE-002 创建邀请码后状态展示

前置条件：未绑定情侣空间。

步骤：

1. 点击创建情侣空间/生成邀请码。

预期结果：

- 显示已绑定状态。
- 展示邀请码。
- 展示成员数。
- 展示上次同步时间或同步状态。
- 文案说明哪些数据会同步，哪些仅保存在本机。

### TC-FE-003 刷新空间状态

前置条件：已创建或已加入情侣空间。

步骤：

1. 点击“刷新空间状态”。

预期结果：

- 页面状态刷新成功。
- 成员数、邀请码、绑定状态与后端一致。
- 无错误时不显示最近错误。

### TC-FE-004 同步错误展示

前置条件：前端已绑定空间。

步骤：

1. 停止后端服务。
2. 在共享空间页触发同步或刷新。

预期结果：

- 页面显示最近错误。
- 本地页面不崩溃。
- 本地愿望、回忆、每日计划仍可操作。

### TC-FE-005 退出情侣空间

前置条件：已绑定情侣空间。

步骤：

1. 点击“退出情侣空间”。
2. 确认退出。

预期结果：

- 绑定状态变为未绑定。
- 本机已有愿望、回忆、每日记录仍保留。
- 不再自动使用旧情侣空间同步。

## 8. 手工双设备联调用例

### TC-E2E-001 A 创建空间，B 加入空间

前置条件：

- 使用两个浏览器环境模拟 A、B，例如 Chrome 普通窗口和无痕窗口。
- 后端服务在线。

步骤：

1. A 打开应用并创建情侣空间。
2. A 复制邀请码。
3. B 打开应用并输入邀请码加入。
4. A、B 分别刷新空间状态。

预期结果：

- A、B 显示相同情侣空间。
- 成员数显示为 2。

### TC-E2E-002 A 新增愿望，B 查看

前置条件：TC-E2E-001 已完成。

步骤：

1. A 新增愿望“周末去看展”。
2. A 手动同步。
3. B 手动同步。
4. B 打开愿望页和首页。

预期结果：

- B 愿望页出现“周末去看展”。
- B 首页对方动态出现 A 新增愿望。

### TC-E2E-003 B 完成愿望，A 查看

前置条件：TC-E2E-002 已完成。

步骤：

1. B 完成“周末去看展”。
2. B 选择“稍后”或“记录回忆”。
3. B 手动同步。
4. A 手动同步。

预期结果：

- A 愿望页中该愿望状态变为已完成。
- A 首页对方动态出现 B 完成愿望。

### TC-E2E-004 A 回答每日问题，B 查看 TA 的回答

前置条件：A、B 已绑定同一个情侣空间。

步骤：

1. A 在首页回答今日问题。
2. A 手动同步。
3. B 手动同步。
4. B 打开首页。

预期结果：

- B 首页“TA 的回答”显示 A 的回答。
- B 未回答时，“我的回答”显示未回答提示。

### TC-E2E-005 解绑后本地数据保留

前置条件：A、B 已绑定同一个情侣空间，B 本机已有愿望和回忆。

步骤：

1. B 点击退出情侣空间。
2. B 查看愿望页、回忆页、首页。

预期结果：

- B 不再显示已绑定情侣空间。
- B 本机愿望和回忆仍存在。
- B 本机仍可继续新增本地数据。

## 9. 验收标准

本轮改动验收通过需满足：

- H5 构建通过。
- 后端 Maven test 通过。
- A/B 创建、加入、状态查询、同步、解绑核心链路通过。
- 未绑定情侣空间时本地功能不受影响。
- 后端不可用时本地操作不阻塞，并能显示同步错误。
- 旧本地数据迁移后不丢愿望、回忆、每日回答。
- 根目录重复前端源码入口已移除，开发只改 `src/`。
